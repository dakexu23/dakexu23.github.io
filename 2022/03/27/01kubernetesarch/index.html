Kubernetes原理及架构设计

发表于 2022-03-28 更新于 2022-03-31

Kubernetes是什么

Kubernetes 是一个可移植的，可扩展的开源平台，用于管理容器化的工作负载和服务，方便了声明式配置和自动化。它拥有一个庞大且快速增长的生态系统。Kubernetes 的服务，支持和工具广泛可用。

Kubernetes能做什么？

  • 敏捷的应用创建和部署，与使用VM相比
  • 关注开发和运维分离；在构建/发布时创赢应用程序镜像，从而将应用程序与基础架构分离。
  • 持续开发、集成和部署
  • 跨开发、测试、生产的环境一致性
  • 跨云的可移植性
  • 资源隔离
  • 资源高效和高密度应用
  • 可观测性
  • 散耦合、分布式、弹性、解放的微服务（服务治理却由istio来完成）

Kubernetes集群的工作原理

控制平面
  1. kube-apiserver：提供HTTP RESTful API接口的主要服务，是Kubernetes里所有资源进行增删改查等操作的唯一入口，也是集群控制的进程。
  2. etcd：kubernetes所有集群数据的数据库
  3. kube-scheduler：负责资源调度的进程。
  4. kube-controller-manager：所有资源对象的自动化控制中心，可以理解为资源对象的”大总管“
Node组件
  1. kubelet：负责对Pod对应容器的创建、启动等任务，与master协作，实现集群管理的基本功能。
  2. kube-proxy：是集群中每个节点上运行的网络代理，实现kubernetes服务（Service）概念的一部分。kube-proxy维护节点上的网络规则，这些规则允许从集群内部或者外部的网络会话与Pod进行网络通信。
插件
  1. DNS：集群 DNS 是一个 DNS 服务器，和环境中的其他 DNS 服务器一起工作，它为 Kubernetes 服务提供 DNS 记录。Kubernetes 启动的容器自动将此 DNS 服务器包含在其 DNS 搜索列表中。
  2. Dashboard：是 Kubernetes 集群的通用的、基于 Web 的用户界面。 它使用户可以管理集群中运行的应用程序以及集群本身并进行故障排除。
  3. 容器资源监控：将关于容器的一些常见的时间序列度量值保存到一个集中的数据库中，并提供用于浏览这些数据的界面。
  4. 集群日志：机制负责将容器的日志数据 保存到一个集中的日志存储中，该存储能够提供搜索和浏览接口。
容器运行时

Kubernetes 支持容器运行时，例如 Docker、 containerd、CRI-O 以及 Kubernetes CRI (容器运行环境接口) 的其他任何实现。

Kubernetes项目的全局架构

  1. K8S中kubelet组件负责与容器运行时进行交互，但并不关心k8s具体是用什么容器运行时；
  2. 具体的容器运行时则通过OCI这个容器运行时规范同底层的Linux操作系统进行交互；
  3. kubelet组件调用网络插件和存储插件分别容器配置网络和持久化存储；
